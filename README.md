# Sentinel AI Incident Management System

## Overview
This project is a database system designed to record and manage behavioral incidents involving advanced AI robots developed by Sentinel Dynamics. These robots operate across healthcare, military, security, mining, childcare, luxury assistance, and research environments.

The system ensures tracking of incidents for safety monitoring, and behavioural analysis.

The full system requirements, business rules, and reporting specifications are provided in the project specification document included in this repository.

---

## Purpose
The goal of this system is to:
- Record AI behavioral incidents in a structured format
- Track robot performance and risk over time
- Support safety compliance and monitoring
- Identify high risk robots and recurring incidents

---

## Expected Files

This project is expected to include the following components:

1. **Database Setup**
   - Creates and configures the database environment

2. **Tables**
   - Defines the required table

3. **Data**
   - Inserts data into the table

4. **Reports**
   - Contains SQL queries used to generate required reports
  
  ## SQL Requirements

- Appropriate use of **primary key** constraints for unique identification
- Use of **not null** constraints where values are mandatory
- Use of **check constraints** to enforce valid ranges
- Use of **unique constraints** where required
- Proper handling of **null values** where data may be unavailable
- Consistent and appropriate **data types** for all fields
- Enforcement of all business rules defined in the project specification
